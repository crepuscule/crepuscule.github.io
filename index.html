3.1 Namespace using Declarations

在学习string，vector等库类型之前，需要先了解命名空间机制。使用声明的命名空间（namespace using declarations）即使用using声明简化了在例如std::cin前面的引用库名声明前缀。而它解决的本质问题是程序中的名称冲突。
一个using声明形式为：using namespace::name;

注意:
程序中所有的库中的名称均要给予using声明
在C++头文件中不要使用using声明，这样会导致源文件在引用头文件时命名空间有冲突。

LVM (Logical Volume Manager)

LVM即Logical Volume Manager（逻辑卷管理），是linux下对磁盘管理的一种机制。与普通磁盘分区管理机制不同的是，它可以在零停机前提下实现对分区大小的调整，其分区不需要必须连续的空间，所以可以跨越不同的磁盘和分区。

首先需要了解一些LVM术语：

  • 物理卷 PV （Physical Volume）
    指磁盘分区或从逻辑上与磁盘分区具有同样功能的设备（如RAID），是LVM的基本存储逻辑块，但和基本的物理存储介质（如分区、磁盘等）比较，却包含有与LVM相关的管理参数
  • 物理块 PE （Physical Extent）
    每一个物理卷PV被划分为称为PE（Physical Extents）的基本单元，具有唯一编号的PE是可以被LVM寻址的最小单元。PE的大小是可配置的，默认为4MB。所以物理卷（PV）由大小等同的基本单元PE组成
  • 逻辑块 LE （Logical Extent）
    逻辑卷LV也被划分为可被寻址的基本单位，称为LE。在同一个卷组中，LE的大小和PE是相同的，并且一一对应
  • 卷组 VG （Volume Group）
    类似于非LVM系统中的物理磁盘，其由一个或多个物理卷PV组成。可以在卷组上创建一个或多个LV（逻辑卷）
  • 逻辑卷 LV （Logical Volume）
    类似于非LVM系统中的磁盘分区，逻辑卷建立在卷组VG之上。在逻辑卷LV之上可以建立文件系统（比如/home或者/usr等）
    LVM就是允许不连续的PV组成一个逻辑上的磁盘（即VG），然后在VG上实现类似非LVM的磁盘分区，显然这样在逻辑上划分分区就更加灵活。

C中define的用法

  1. 简单用法
    #define MAX 5+5
    在预编译阶段在程序中发现MAX即将其替换为5+5, 不会对宏体做任何运算。
    如果有语句int a=NN;a的值应为5+55+5=35
  2. 带参数的宏定义
    #define add(x,y) x+y
    如果要用define实现一个加法函数，上述做法是错误的，正确做法是：#define add(x,y) ((x)+(y)) 如果用宏实现类似函数的功能，必须将每一个参数和整个宏体都加上括号。
  3. 多行宏定义
    如同多数命令行一样，在行末尾加一个反斜杠”\”，即可在下一行继续书写宏定义。
  4. 宏中三个符号
  • x##y 表示连接 #define con(x,y) x##y int n=con(4,5); n的值为45
  • #@x 表示给x加上’’,使其变为一个const char
  • #x 表示给x加上””,使其变为一个字符串

1.1编译器

编译器是一种程序，它读入某种源语言编写的程序，将其翻译成一个与之等价的以另一种目标语言编写的程序

编译的分析-综合模型

分析部分将源程序分为基本块并形成中间表示，综合部分将中间表示转换为目标程序。综合部分技术性更强。在分析中，源程序蕴含的操作被确定为语法树，树的一个节点表示操作，它的子节点表示该操作的参数。
